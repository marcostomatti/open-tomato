Git repository was already initialized before this plan began. The .git directory exists with remote origin configured and branch feature/ODT-1-plan-bootstrap.
ESLint config uses .eslintrc.json (legacy format) with `plugin:@typescript-eslint/recommended-type-checked`; `parserOptions.project: true` enables auto-discovery of per-directory tsconfigs for type-aware linting. The preload override sets both `node: true` and `browser: true` since preload scripts bridge both environments.
Root package.json uses npm workspaces with `"workspaces": ["src/main", "src/renderer", "src/preload", "src/shared"]`. ESLint v8 (not v9) is required because the project uses the legacy .eslintrc.json config format; ESLint v9 uses a different flat config format. @typescript-eslint v7 is compatible with ESLint v8. The existing project tooling scripts (tasks, tasks:next) and their devDeps (@linear/sdk, inquirer, @types/bun) were preserved alongside the new Electron app devDeps.
TypeScript composite projects (src/main, src/renderer, src/preload, src/shared) do not require explicit `outDir` — tsc --build succeeds with the current tsconfig setup. The sub-tsconfigs for main/renderer/preload reference `../shared` via `"references"` and use `paths` aliases `@shared/types` and `@shared/ipc` pointing to `../shared/types/index.ts` and `../shared/ipc/index.ts` respectively.
